### Ⅰ. 文件操作 (File I/O)

文件操作是程序与磁盘数据交互的基础。

#### 1. 核心操作模式

| 模式 | 描述 | 文件不存在时 | 文件已存在时 | 风险提示 |
| :--- | :--- | :--- | :--- | :--- |
| **`r`** | 读取 (Read) | 抛出 `FileNotFoundError` | 指针在开头 | 只能读，不能写 |
| **`w`** | 写入 (Write) | **创建**新文件 | **清空并覆盖**原有内容 | **慎用**，容易丢失数据 |
| **`a`** | **追加 (Append)** | **创建**新文件 | 指针在**文件末尾** | 适合记录日志、末尾添加数据 |

#### 2. 上下文管理器 (`with open`)

专业编程中**强烈推荐**使用 `with open` 来处理文件操作，以确保资源安全。

| 语法 | 优势 |
| :--- | :--- |
| `with open(filename, mode) as f:` | **自动关闭文件**：无论 `with` 块中代码是否运行成功，文件都会被自动安全关闭，防止资源泄漏或数据损坏。 |

**代码示例 (安全追加)：**

```python
# 'a' 模式确保在文件末尾添加内容
with open("log.txt", "a") as f:
    f.write("新的一行日志。\n")
````

### Ⅱ. 错误与异常处理 (Exception Handling)

异常处理是让你的程序在遇到错误时能够**优雅地恢复**而不是崩溃的关键技能。

#### 1\. `try...except` 结构

| 结构块 | 作用 | 流程 |
| :--- | :--- | :--- |
| **`try:`** | 放置**可能引发异常**的代码。 | Python 尝试执行这里的代码。 |
| **`except ExceptionType:`** | 放置当 `try` 块中发生**特定异常**时要执行的处理代码。 | 如果发生异常，程序跳到这里执行，然后继续向下运行。 |

#### 2\. 常见异常及其处理

| 异常类型 | 触发原因 | 示例场景 |
| :--- | :--- | :--- |
| **`ZeroDivisionError`** | 尝试将数字除以零。 | `10 / 0` |
| **`FileNotFoundError`** | 尝试读取或访问一个不存在的文件。 | `open("missing.txt", "r")` |
| **`TypeError`** | 对不兼容的数据类型执行操作（如数字和字符串相加）。 | `5 + "text"` |

**代码示例 (处理文件缺失)：**

```python
try:
    with open("config.json", "r") as f:
        data = f.read()
except FileNotFoundError:
    print("错误：配置文件未找到，正在使用默认设置。")
    data = "{}" # 赋予一个默认值，防止程序崩溃
```
